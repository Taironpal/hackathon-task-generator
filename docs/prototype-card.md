## 1. Название проекта

**Сверка** - ИИ-генератор вариантов заданий

## 2. Кейс

**Кейс 4. ИИ-генератор вариантов заданий** (СберОбразование + Школа 21, хакатон «ИИ для образования», 18.05 - 02.06.2026)

## 3. Краткое описание (до 500 знаков)

Учитель указывает тему, класс и количество задач (можно собрать контрольную из нескольких блоков разных тем и предметов). Сервис подбирает эталонные задачи из библиотеки школьной программы, через GigaChat размечает в них изменяемые величины и формулу ответа, затем sympy подставляет значения и точной арифметикой считает каждый вариант. На выходе два готовых DOCX: для класса с местом под ФИО и ключи учителя с ответами и подставленными значениями.

## 4. Проблема, которую решает

Учитель тратит **~40 минут** на ручную подготовку 4 вариантов контрольной работы (переписывает условия, меняет числа, считает ответы). Из-за спешки варианты получаются упрощёнными или содержат ошибки в ответах. Существующие генераторы либо требуют экспертной настройки шаблонов, либо генерируют задачи с нуля без гарантии корректности и соответствия программе.

## 5. Целевая аудитория

- **Основная:** учителя математики 1-11 класса
- **Расширение:** учителя точных наук с численными ответами (физика, химия, информатика - в&nbsp;MVP уже есть)
- **Через интеграции:** методические объединения, репетиторы, авторы дидактических материалов

## 6. Ключевые функции (закрывают бриф Кейса 4)

| Функция | Реализация |
|---|---|
| Автоматический анализ эталонного задания | GigaChat + sympy-валидация формулы при анализе |
| Выделение изменяемых и неизменяемых частей | модель Slot со свойством `locked`, формула ответа из слотов |
| Учёт уровня сложности | диапазоны на каждый слот + эвристика сложности 1-5 на каждый сгенерированный вариант |
| Настройка типов изменений | пять типов слотов: int, decimal, percent, fraction, string |
| Запрет правки отдельных частей | `Slot.locked = True` уважается генератором |
| Сравнение вариантов по сложности | оценка 1-5 для каждого варианта, видна на карточке |
| Редактирование результата | редактируемый диапазон слотов и блоков до генерации |
| Выгрузка готового комплекта с разметкой и ответами | два DOCX через python-docx: «варианты для учеников» + «ключи учителя» |

**Сверх ТЗ:**
- конструктор из нескольких блоков (разные темы и предметы в одной контрольной)
- 96 эталонов из 12 учебников программы 1-11 класса
- 4 предмета: математика, физика, химия, информатика
- stem-поиск по русской морфологии («обыкновенные дроби» → «сложение обыкновенных дробей»)
- согласование существительных по числу («1 час / 2 часа / 5 часов»)

## 7. Технологический стек

- **Backend:** Python 3.12, FastAPI, uvicorn
- **LLM:** GigaChat (`GigaChat`, она же Lite в API) - обязательно для бонусов хакатона
- **Валидация:** sympy с `nsimplify(rational=True)` - точная арифметика без float-артефактов
- **Морфология русского:** собственный словарь ~50 школьных существительных + правило согласования по числу + stem-поиск по 20 типовым окончаниям
- **Библиотека:** 96 эталонов в JSON с привязкой к учебникам (Виленкин, Макарычев, Атанасян, Алимов, Перышкин, Мякишев, Габриелян, Босова, Моро)
- **Экспорт:** python-docx
- **Frontend:** Next.js 16 (App Router), TanStack Query, Tailwind 4, адаптивный hero + sticky footer
- **Хранилище:** in-memory dict с UUID (в демо), sqlite/PostgreSQL для прода

## 8. Архитектурное преимущество - sympy-валидация

Каждый сгенерированный ответ считается формулой и перепроверяется sympy. На стенде поймана реальная галлюцинация GigaChat: LLM указала ответ для `(48,6 + 29,04) × 2` как 136,28 вместо корректных 155,28. Система зафиксировала расхождение, использовала формулу, во всех сгенерированных вариантах ответы оказались правильными. **First-pass rate sympy на проверенных кейсах: 100%.**

## 9. Сценарий использования (для демо)

1. Учитель открывает главную - короткий hero и кнопка «Открыть конструктор»
2. В конструкторе видит карточку блока 01: выбор предмета (математика по умолчанию), поле темы с автодополнением из библиотеки и чипами популярных тем для выбранного класса
3. Например: блок 01 - «обыкновенные дроби», 6 класс, 2 задачи
4. Жмёт «Добавить блок» и собирает второй: «плотность вещества», физика, 7 класс, 2 задачи
5. Указывает общее количество вариантов - 4
6. Жмёт «Сгенерировать». За 30-60 секунд: подбор эталонов из библиотеки → GigaChat размечает слоты → sympy валидирует каждый из 16 вариантов
7. Открывается страница результатов: 4 варианта по 4 задачи, в каждом подзаголовки блоков с темой и классом, ответы и метка sympy ✓ на каждой задаче
8. Скачивает «Варианты .docx» (для класса, с местом под ФИО и ответ) и «Ключи .docx» (с ответами, подставленными значениями, sympy-метками)

**Хронометраж:** 40 минут ручной работы → 30 секунд автоматически.

## 10. Метрики MVP (проверено на стенде)

- **First-pass rate sympy:** 100% на смешанной контрольной (16 задач: математика 6 + физика 7), 100% на одно-предметных кейсах
- **Время полного цикла** (анализ + 4 варианта на 1 блок): 8-12 секунд; на 2 блока: 30-60 секунд
- **Покрытие программы:** 96 эталонов из 12 учебников, 1-11 класс, 4 предмета
- **Поддержка русского:** морфологическое согласование «1 час / 2 часа / 5 часов» для ~50 школьных существительных, stem-поиск по библиотеке устойчив к падежам
