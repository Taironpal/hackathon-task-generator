# Сверка - ИИ-генератор вариантов заданий

Прототип для хакатона **«ИИ для образования: автоматизация рутинных задач»** (СберОбразование + Школа&nbsp;21, 18.05 - 02.06.2026). **Кейс&nbsp;4: ИИ-генератор вариантов заданий.**

Учитель выбирает **тему, класс и количество задач** (можно собрать контрольную из нескольких блоков разных тем и предметов) → сервис подбирает эталонную задачу из библиотеки программы, размечает в ней изменяемые величины через GigaChat → **генерирует N эквивалентных вариантов**, у каждого ответ перепроверен sympy → выгрузка двух готовых DOCX: для класса и для учителя.

```
┌───────────────────┐   library   ┌──────────────┐   GigaChat   ┌──────────────┐   sympy    ┌────────────┐  python-docx ┌────────────┐
│ конструктор блоков│────────────▶│  эталон      │─────────────▶│ ReferenceTask│───────────▶│ N вариантов│─────────────▶│ DOCX × 2   │
│ тема/класс/кол-во │  search     │  из программы│  analyze     │  + slots     │ generate   │  (sympy ✓) │              │ класс +    │
│ 4 предмета        │  + stem     │  (96 эталонов)│             │  + formula   │ _variants  │            │              │ учитель    │
└───────────────────┘             └──────────────┘              └──────────────┘            └────────────┘              └────────────┘
```

## Что закрывает в брифе

| Требование Кейса 4 | Где у нас |
|---|---|
| Автоматически анализирует эталонное задание | `backend/llm/analyzer.py` + промпт `SYSTEM_ANALYZER` |
| Выделяет изменяемые и неизменяемые части | модель `Slot` с полем `locked`, формула ответа из слотов |
| Учитывает уровень сложности | `Slot.range`, эвристика `_score_difficulty` 1-5 |
| Генерирует нужное количество вариантов | `backend/variant_generator.py` |
| Настройка типов изменений | `SlotType: int / decimal / percent / fraction / string` |
| Запрет правки отдельных частей | `Slot.locked = True`, уважается генератором |
| Сравнение вариантов по сложности | `GeneratedVariant.difficulty` на карточке варианта |
| Выгрузка комплекта с разметкой и ответами | `/assignments/{id}/students.docx` + `/teacher.docx` |

Дополнительно (сверх ТЗ):
- **sympy-валидация** каждого ответа из формулы - защита от LLM-галлюцинаций. Расхождение `original_answer` LLM с формулой логируется и не блокирует генерацию, потому что формула важнее.
- **Конструктор из нескольких блоков**: одна контрольная может содержать задачи разных тем и предметов (например, обыкновенные дроби + плотность вещества).
- **4 предмета**: математика, физика, химия, информатика. 96 эталонов из 12 учебников программы.
- **Морфологический stem-поиск** по библиотеке: «обыкновенные дроби» находит «сложение обыкновенных дробей».
- **Согласование существительных по числу** в русском («1 час / 2 часа / 5 часов») для ~50 школьных слов.

## Быстрый старт

### 1. Backend (FastAPI, порт 8000)

```bash
# из корня репозитория
python -m venv .venv
.venv\Scripts\activate            # Windows
# source .venv/bin/activate       # Linux / macOS
pip install -r requirements.txt

# скопировать .env.example в .env и вписать свой GigaChat Authorization Key
copy .env.example .env            # Windows
# cp .env.example .env            # Linux / macOS

# запустить API
python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000
```

Проверка:
```bash
curl http://127.0.0.1:8000/health
# {"status":"ok","worksheets":0}
```

### 2. Frontend (Next.js, порт 3000 или 3001)

```bash
cd web
npm install
npm run dev
```

Открыть **http://localhost:3000/** - главная с описанием продукта, либо **http://localhost:3000/reference** - конструктор контрольной.

> Если 3000 занят, Next автоматически берёт 3001. CORS уже разрешает оба адреса.

### 3. CLI-смоук (без UI)

```bash
# проанализировать одну задачу
python -m scripts.analyze "В магазине было 120 кг яблок. За день продали 25%. Сколько кг осталось?" --grade 5

# end-to-end: эталон → 4 варианта
python -m scripts.variants "Велосипедист проехал 24 км за 2 часа. С какой средней скоростью он ехал?" --grade 6 --count 4

# smoke test на 3 встроенных эталонах
python -m scripts.variants --count 3
```

## Демо-флоу

1. Открыть [http://localhost:3000/](http://localhost:3000/) - главная с CTA «Открыть конструктор»
2. Перейти в [/reference](http://localhost:3000/reference) - конструктор контрольной
3. В блоке 01 выбрать предмет (математика по умолчанию), указать класс и тему. Можно ткнуть в чип популярной темы для выбранного класса.
4. Указать количество задач в варианте.
5. Опционально нажать «Добавить блок» и собрать смешанную контрольную (например, математика 6 кл + физика 7 кл).
6. Указать общее количество вариантов (по умолчанию 4) и нажать «Сгенерировать».
7. Через 30-60 секунд (LLM-анализ + генерация + sympy-валидация) открывается страница с готовыми вариантами. У каждой задачи виден ответ, подставленные значения слотов и метка sympy ✓.
8. Скачать «Варианты .docx» (для учеников, с местом под ФИО и ответ) и «Ключи .docx» (для учителя, с ответами, подставленными значениями и пометками sympy).

## API (REST)

| Метод | Эндпоинт | Что делает |
|---|---|---|
| POST | `/compose-assignment` | блоки `[{topic, grade, subject, tasks_per_variant}]` + `variants_count` → контрольная с N вариантами |
| GET  | `/assignments/{id}` | получить ранее собранную контрольную |
| GET  | `/assignments/{id}/students.docx` | DOCX для печати ученикам |
| GET  | `/assignments/{id}/teacher.docx`  | DOCX с ответами и метаинфо для учителя |
| GET  | `/library?query=...&grade=...&subject=...` | поиск по библиотеке эталонов (96 задач, stem-поиск) |
| POST | `/analyze` | текст эталона → `ReferenceTask` (template, slots, answer_formula) |
| POST | `/generate-variants` | `ReferenceTask` + count → `VariantSet` |
| POST | `/quick-generate` | быстрый путь: тема + класс + count → варианты в одном вызове |
| GET  | `/health` | живой ли сервис |

Swagger UI: `http://127.0.0.1:8000/docs`

### Пример: собрать контрольную из двух блоков

```bash
curl -X POST http://127.0.0.1:8000/compose-assignment \
  -H "Content-Type: application/json; charset=utf-8" \
  -d '{
    "variants_count": 4,
    "blocks": [
      {"subject": "математика", "grade": 6, "topic": "обыкновенные дроби", "tasks_per_variant": 2},
      {"subject": "физика", "grade": 7, "topic": "плотность вещества", "tasks_per_variant": 2}
    ]
  }'
```

## Защита от LLM-галлюцинаций (наш козырь)

GigaChat иногда возвращает правильную формулу, но неверный `original_answer`. Анализатор сверяет ответ из LLM с результатом формулы через sympy: расхождение логируется, в дальнейшей генерации используется формула, sympy на каждом варианте подтверждает корректность. **First-pass rate sympy на проверенных кейсах: 100%.**

Пример: задача `(48,6 + 29,04) × 2`. LLM написала ответ 136,28. Sympy посчитал 155,28. Расхождение зафиксировано, во всех 4 сгенерированных вариантах ответы корректны.

## Известные ограничения

- **In-memory store** - после рестарта `WORKSHEETS`, `VARIANT_SETS`, `COMPOSITE_ASSIGNMENTS` чистятся. Для прода нужен sqlite/postgres (1 день работы).
- **GigaChat Freemium** - лимит 900 тысяч токенов до 15.05.2027.
- **Морфология русского** - покрыто ~50 школьных существительных, для остальных без согласования по числу.
- **Темы вне библиотеки** - LLM может проанализировать любую вставленную задачу через `/analyze`, но `/compose-assignment` пока работает только с темами из библиотеки.
